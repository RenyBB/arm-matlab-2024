Introduction to PTB

In this section, well cover the basics of PsychToolbox.


In the following set of exercises, well build up a simple experiment.

This first step is the set-up phase.

The set-up phase will include several steps. For example, you may need to:

  • Generate or enter a unique code for each participant:

  • Determine which keyboard will be used for collecting responses (GetKeyboardIndices)

  • Determine what the stimuli will be, and load them up into a structure (which youll loop over in the main loop).

  • Determine which screen will be used for displaying the stimuli (Screen('Screens')).

  • Open up a window where youll show the stimuli/instructions to the participant (Screen('OpenWindow') or PsychImaging('OpenWindow'))

  • Show some instructions on the screen so that the participant knows what to do

The next thing well do is set up our trial structure.

Our task will be a word discrimination task where participants will press one key when they see the word DOG and another key when they see the word CAT.

For each trial, well need to know the following information:

  1. The stimulus that will be presented

  2. The expected response

And well need to be able to store:

  1. The actual response

  2. The reaction time.

We might also want to store other information, like accuracy, if were going to need it for our analysis.

Were now on to the main part of the experiment. In this part, well loop over the trials and display the words DOG or CAT on the appropriate trials.

On each iteration of the loop, well have to do the following:

  1. Select a stimulus from the trial structure

  2. Show a fixation cross

  3. Display the stimulus

  4. Collect a response.

But before we do all this, well need to know how many times to loop. Well use the size/length function to set up our loop. We could do something like the following:

  • On each iteration of the loop, the value of t will be set to 1, then 2, then 3, then 4, until it reaches the value defined by size(trialTable,1) (i.e., 40)

  • Well use the value of t to select a stimulus from the trialTable, and to input values (e.g., response times) into the trialTable

Ex

Before we can start putting code inside the main trial loop, well still need to add a little more code which will help us to collect responses.

Well use KbQueueCheck() to collect the responses because we want to have accurate timing.

The steps for using KbQueueCheck() (from the slides) are:

  1. Create the queue: KbQueueCreate(DEVICE_TO_USE)

  2. Start the queue: KbQueueStart(DEVICE_TO_USE)

  3. Flush the queue: KbQueueFlush(DEVICE_TO_USE)

  4. Check the queue: KbQueueCheck(DEVICE_TO_USE)

  5. Stop the queue: KbQueueStop(DEVICE_TO_USE)

  6. Release the queue: KbQueueRelease(DEVICE_TO_USE)

Steps 1 & 2 will be before the loop, 3 & 4 inside the loop, and 5 & 6 after the loop.
